Release Notes

Change Notes

  • 0.9.203

    April 7, 2026
    • Added a new phone remote web app for TAPE-16 with track arm controls, timecode, marker controls, rec safe, and transport controls.
    • Added outboard send/return routing within S1-S4.
    • You can now right click to rename sends S1-S4.
    • Added activation/license overlay improvements and more calibration controls.
    • Added scrub function.
    • Hold Shift while FW and RW to run at 10x speed.
    • Added an Always open in fullscreen option.
    • Improved saved windowed mode behavior and window restore handling.
    • Fixed sidebar button sizing so it affects the correct controls.
    • Improved VU height behavior and reel/meter layout spacing.
    • Refined timecode scaling, shadows, and several UI layout details.
    • Improved controller setup and MIDI routing workflow.
    • Added DAW controller management improvements. More controllers to be added.
    • Fixed 16/24 bit silent playback issue.
    • Added a 7 day trial with all features instead of demo mode.
    • Improved bounce/export behavior and FF/RW audio behavior.
    • Improved crash reporting, session diagnostics, and legacy routing handling.
    • Refreshed various graphics, reel controls, splash visuals, and plugin window UI.
    • Various audio interface issues patched.
    View release on GitHub

  • 0.9.202

    March 11, 2026
    • Expanded mono-first tape workflow: Audio I/O now clearly communicates that tape tracks are mono by design.
    • Changed channel presentation from paired routing to single-channel options where appropriate, matching tape-machine behavior.
    • Added Print Instrument To Tape As modes for MIDI/instrument commit control: MONO SUM (L+R), LEFT ONLY, RIGHT ONLY.
    • Improved Audio Settings panel behavior with stronger layout anchoring and reduced overlap risk at smaller window sizes.
    • Refined transport button-state feedback for range-dependent realtime operations.
    • Added/updated latency visibility in diagnostics workflows to better reflect real recording and routing scenarios.
    • Reduced audible clicks on stop and solo/mute transitions with additional output smoothing and transition handling.
    • Improved crash-recovery guidance so plugin-related failures point users to clear cache + Deep Scan steps.
    View release on GitHub

  • 0.9.201

    March 9, 2026
    • Published macOS build for 0.9.201.
    • Added crash-recovery load safeguards: one-shot safe mode, auto-exit on loading another tape, and explicit Exit Recovery Mode action.
    • Improved crash report bundles with broader crash file capture, stronger app-log inclusion, and richer diagnostics metadata.
    View release on GitHub

  • 0.9.20

    March 8, 2026
    • Added 3-way tape speed switch (7.5 / 15 / 30) with persisted per-tape speed behavior.
    • Added Mix Out + Single Tracks output routing and cleaned up Audio Settings layout/consistency.
    • Improved crash reporting: in-app bug/crash dialog now submits bundled diagnostics with fallback.
    • Applied subtle high-end lift for 7.5 IPS and 15 IPS profiles.
    View release on GitHub

  • 0.9.191

    March 4, 2026
    • Added lane-aware Track FX slots in Plugin Manager: P1, P2, M1, and M2.
    • Updated split track FX labels to PFX/MFX with larger text for better readability.
    • In SENDS mode, restored the original single FX button behavior on track strips.
    • Temporarily hid/disabled Offline Bounce in the bounce panel.
    • Crash recovery now opens a blank recovery tape after unclean shutdown and offers one-click crash report send.
    View release on GitHub
